# Getting Started

The current implementation focuses on the first workflow slice rather than a large skill catalog.

## Think in stages

Do not start from "which folder should I open?"

Start from "which research stage am I in?"

1. Scope the problem.
2. Track recent papers if needed.
3. Synthesize the literature.
4. Turn gaps into a design.
5. Draft the paper.
6. Review and revise.

## Entry point

The default entry skill is `using-paperskills`, which routes requests to the right downstream skill.

## Included in this first implementation slice

- workflow entry skill scaffold
- orchestration skill scaffolds
- shared brief schemas
- updated product-facing documentation
