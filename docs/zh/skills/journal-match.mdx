# journal-match

`journal-match` 用来根据稿件内容、研究方向和相近论文的发表 venue，推荐更适合投稿的目标期刊。

## 适用场景

- 我已经有稿件或摘要，但还没决定投哪里
- 我想知道哪些期刊和我的研究主题更匹配
- 我想同时比较 scope match、影响力和开放获取情况
- 我希望先得到一版分层期刊推荐

## 它会做什么

这个 skill 通常会：

1. 先提取稿件的学科、子领域、方法、关键词和语言
2. 检索相似论文，看它们通常发表在哪些期刊
3. 补充期刊的元数据，例如 h-index、works count、OA 状态
4. 给出分层推荐和匹配理由

## 你最好提供什么输入

- 稿件文件、摘要，或者较清晰的研究主题描述
- 如果你只接受某类期刊，也可以说明范围
- 如果语言不是英文，建议同时说明投稿语言目标

## 常见用法

下面示例统一使用显式 skill 前缀，便于 Agent 稳定触发对应能力。

### 基于摘要推荐期刊

```text
/journal-match Please recommend target journals for this manuscript abstract.
```

### 指定中文说明

```text
/journal-match 请根据这篇关于 AI 搜索广告的论文摘要，推荐适合投稿的期刊，并用中文解释匹配理由。
```

## 输出通常包含什么

- 分层期刊推荐
- 每个期刊的 scope match 解释
- 相似论文样本
- 基本影响指标和 OA 状态
- 后续投稿准备建议

## 使用时要注意

- 它给的是基于公开数据和相似论文分布的建议，不是编辑部真实录用判断
- 如果你的稿件过短，推荐结果会更依赖主题关键词
- 非英文或非常小众期刊的公开元数据覆盖可能有限

## 推荐搭配流程

一般在稿件已经比较成型之后使用。你可以先用 `/abstract` 整理摘要，再用 `/journal-match` 做投刊筛选；投稿前再继续用 `/peer-review` 和 `/cite-verify`。
