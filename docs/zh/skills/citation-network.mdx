它不是从零开始找文献的入口，更适合你已经拿到一篇或几篇种子论文后，再顺着引用关系去理解领域结构。

## 适用场景

- 我已经有几篇关键论文，想看它们在领域中的位置
- 我想找多个种子论文共同引用或共同被引的核心文献
- 我想识别 foundational works、recent frontier 和 bridge papers
- 我需要一个可视化引文网络报告

## 它会做什么

这个 skill 通常会：

1. 解析 DOI 或论文标题，先定位种子论文
2. 拉取种子论文的 references 和 citations
3. 建立一层到两层深度的引文网络
4. 识别高连接节点、聚类和潜在桥梁论文
5. 生成带网络图和时间线的 HTML 报告

## 你最好提供什么输入

- 一篇或多篇种子论文的 DOI
- 如果没有 DOI，也可以提供准确标题
- 如果你关心特定语言或输出形式，也可以提前说明

## 常见用法

下面示例统一使用显式 skill 前缀，便于 Agent 稳定触发对应能力。

### 用 DOI 构建网络

```text
/citation-network 10.1038/s41586-023-06221-2
```

### 多篇种子论文联合分析

```text
/citation-network 10.1038/s41586-023-06221-2 + Attention is All You Need
```

## 输出通常包含什么

- 引文网络图
- 核心论文列表
- 主题簇或方法簇说明
- foundational works 与 recent frontier
- 简要统计信息

## 使用时要注意

- 这个任务依赖外部公开学术 API，结果覆盖面受数据源限制
- 如果种子论文过多，网络会很快膨胀，通常需要限制深度
- 它关注的是“论文之间如何连接”，不是完整综述写作

## 推荐搭配流程

如果你还没有种子论文，先用 `/lit-search`；如果你已经从网络中识别出一个潜在空白，再用 `/research-gap` 做进一步分析。
