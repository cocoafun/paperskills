# research-gap

`research-gap` 用来分析一个领域的研究地貌，并识别时间、方法、主题、应用或人群层面的潜在研究缺口。

## 适用场景

- 我想知道某个方向还有哪些值得做的问题
- 我需要把“感觉有空白”变成更可论证的 gap statement
- 我想区分 temporal gap、methodological gap、thematic gap 等不同类型
- 我需要从文献趋势出发生成潜在研究问题

## 它会做什么

按照当前 skill 设计，这个能力通常会：

1. 对目标主题做较大范围的文献地貌扫描
2. 收集年度趋势、核心论文、主要 venue、关键作者和相关概念
3. 从多个维度识别潜在缺口
4. 给出每个 gap 的证据、可行性和优先级
5. 生成结构化报告和建议研究问题

## 你最好提供什么输入

- 一个尽量明确的研究主题
- 如果你有学科、方法或应用场景边界，也建议说明
- 如果你只想看最近几年，也可以补充时间窗口

## 常见用法

下面示例统一使用显式 skill 前缀，便于 Agent 稳定触发对应能力。

### 分析一个主题的研究空白

```text
/research-gap Analyze research gaps in LLM-assisted academic writing.
```

### 指定应用场景

```text
/research-gap 请分析 AI 搜索广告研究中的方法缺口、应用缺口和近三年的新问题。
```

## 输出通常包含什么

- 领域地貌概览
- 时间趋势
- 主要主题簇
- 多类型 research gaps
- 每个 gap 对应的研究问题建议
- 优先级和可行性判断

## 使用时要注意

- 这是当前成本最高的 skill 之一，因为它需要较大范围的文献地图
- 如果主题过大，它通常需要你先缩小问题范围
- 它关注“缺口识别”，不替代基础检索

## 推荐搭配流程

最好先用 `/lit-search` 获得基础文献池；如果你已经有几个关键种子论文，也可以先用 `/citation-network` 帮助定位，再转入 `/research-gap`。
