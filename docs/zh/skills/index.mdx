# 技能介绍

当前 `paperskills` 仓库实际包含 9 个子 skill：

- `topic-framing`
- `lit-search`
- `abstract`
- `cite-verify`
- `citation-network`
- `research-gap`
- `peer-review`
- `paper-tracker`
- `journal-match`

为了提高 skill 触发的稳定性，实际使用时建议在提示词前显式加上对应前缀，例如 `/lit-search`、`/peer-review`、`/paper-tracker`。

## 按任务阶段选择

### 选题与研究定位

- `topic-framing`
  适合把模糊研究想法收敛成可研究的问题和论文标题
- `research-gap`
  适合在一个领域里系统识别研究缺口、提出潜在研究问题

### 检索与文献分析

- `lit-search`
  适合跨多个公开学术源做一次性文献检索、去重和排序
- `paper-tracker`
  适合追踪最近一段时间的新论文、作者、期刊或 venue 动态
- `citation-network`
  适合从种子论文出发构建引文网络、找核心论文和桥梁论文

### 写作与投稿准备

- `abstract`
  适合根据论文正文生成多种版本摘要
- `peer-review`
  适合对草稿做模拟审稿和问题诊断
- `cite-verify`
  适合提交前逐条核验参考文献与引文准确性
- `journal-match`
  适合根据稿件主题、方法和相近论文推荐目标期刊

## 一个常见工作流

很多真实任务会按下面顺序推进：

1. 先用 `/topic-framing` 收敛研究题目
2. 再用 `/lit-search` 找相关文献
3. 如需识别空白，用 `/research-gap` 做缺口分析
4. 如果有种子论文，再用 `/citation-network` 补充网络视角
5. 草稿写完后，用 `/abstract`、`/peer-review`、`/cite-verify` 完成投稿前整理
6. 最后如果还没有明确投稿目标，用 `/journal-match` 做期刊匹配

## 快速判断

### 我还没有题目

用 `/topic-framing`

### 我想先找论文

用 `/lit-search`

### 我只关心最近的新论文

用 `/paper-tracker`

### 我想知道这个领域还有哪些没做

用 `/research-gap`

### 我想看关键论文之间怎么互相引用

用 `/citation-network`

### 我已经有论文草稿

先用 `/peer-review`，再用 `/cite-verify`

### 我只需要摘要

用 `/abstract`

### 我想投稿但不知道投哪里

用 `/journal-match`
