# Design Playbook

Use this reference when turning literature gaps into a study design.

## Core sequence

1. Define the target contribution.
2. Confirm manuscript type and study-completion status.
3. Convert gaps into research questions.
4. Match each question to a method family.
5. Specify data, variables, constructs, or evaluation criteria.
6. State the main risks and assumptions.

## Matching logic

- explanatory question -> empirical design
- causal question -> identification strategy or experiment
- artifact-performance question -> benchmark or systems evaluation
- theory-building question -> qualitative or conceptual design

## Evidence discipline

- Every design claim should be traceable to a gap or limitation in prior work.
- Distinguish what is feasible now from what would require new infrastructure.
- If the design is only one option among many, say why it is preferred.
- If the study has not been run, keep the output in future tense and avoid pseudo-results.

## Failure modes

- proposing a method that does not answer the question
- inflating causal language without identification support
- listing variables without a measurement plan
- claiming significance before the design is credible
- presenting a proposal-style design as if the data have already been collected

## Output priorities

- research question quality
- contribution clarity
- method fit
- feasibility
- main validity threats
