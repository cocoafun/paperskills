# Writing Playbook

Use this reference when converting an evidence set into review prose.

## Default structure

1. Research context
2. Search scope and source coverage
3. Thematic synthesis
4. Research gaps
5. Limitations
6. Future directions

## Synthesis rules

- Write by theme, question, or method family. Do not write paper-by-paper summaries.
- Open each theme with a 1-2 sentence synthesis claim.
- Support the claim with 2-4 representative papers.
- Note disagreement, tradeoffs, or uncertainty before moving to the next theme.

## Claim discipline

- Prefer `The literature suggests` only when multiple papers support the statement.
- Use `One study reports` or `Several recent papers indicate` when evidence is thinner.
- Mark your own reasoning with phrases such as `Taken together` or `This suggests`.
- Never turn a trend inference into a quoted fact.

## Gap analysis

Common gap categories:

- coverage gaps: underexplored tasks, datasets, or populations
- method gaps: weak baselines, poor evaluation, unstable pipelines
- evidence gaps: missing replication, abstract-only claims, few strong studies
- deployment gaps: limited usability, cost, governance, domain adaptation

Prefer concrete gaps over generic statements like `more research is needed`.

## Related-work mode

When the user wants a paper's `related work` section:

- compress to 3-4 themes
- emphasize contrast between prior work and the user's target angle
- keep historical setup short
- end with the exact gap or positioning sentence the user's paper can occupy

## Limitation notes

The draft should say when:

- source coverage is incomplete
- the review window is narrow
- only abstracts or metadata were examined
- preprints were included

## Final checks

- Every paragraph has a purpose.
- Every substantive claim can be traced to evidence.
- The review ends with a sharper problem framing than it started with.
