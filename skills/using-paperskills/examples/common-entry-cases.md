# Common Entry Cases

## Case 1

User request:

`我想做一个关于大模型和科学研究的选题，但还不知道具体方向。`

Recommended stage:

`research-scoping`

Reason:

The request is exploratory and under-scoped.

## Case 2

User request:

`追踪最近两周供应链韧性相关论文，优先 POM 和 Management Science。`

Recommended stage:

`paper-tracker`

Reason:

The request is time-bounded and retrieval-first.

## Case 3

User request:

`基于这些文献帮我写 related work，并总结 gap。`

Recommended stage:

`literature-review`

Reason:

The user needs synthesis, not just retrieval.

## Case 4

User request:

`根据这些 gap 帮我形成研究问题和假设。`

Recommended stage:

`research-design`

Reason:

The work has moved from review to study planning.

## Case 5

User request:

`这是我的论文摘要和方法部分，帮我审稿。`

Recommended stage:

`peer-review`

Reason:

The request is critique of a single paper artifact.

## Case 6

User request:

`请使用 using-paperskills。题目为“人工智能搜索引擎广告模式策略研究”，用中文。完整写一篇论文。`

Recommended workflow:

`research-scoping -> literature-review -> paper-drafting -> manuscript-finalization`

Reason:

The user wants a complete Chinese undergraduate thesis from only a topic and does not provide completed empirical evidence. The workflow still needs topic scoping, literature synthesis, working-draft generation, and a finalization stage so the output is a thesis-ready manuscript rather than only a draft.
