# Finalization Playbook

Use this reference when promoting a draft package to a final manuscript.

## Core sequence

1. Confirm the manuscript type and target artifact.
2. Confirm whether the user wants `draft`, `final manuscript`, or `submission-ready` output.
3. Run a completeness checklist on required sections.
4. Normalize abstract, keywords, introduction, conclusion, and references.
5. Remove draft-only markers, TODOs, duplicated paragraphs, and unsupported overclaims.
6. Re-check evidence boundaries before the final readiness claim.

## Finalization rules

- `paper-drafting` produces a working manuscript. `manuscript-finalization` is the stage that turns it into a finished deliverable.
- Do not use stylistic polish to hide missing logic, missing references, or unsupported claims.
- If evidence is partial, the final manuscript may still be polished, but it must explicitly retain the limitation language.
- Keep title, abstract, chapter titles, and conclusion mutually consistent.
- Ensure the conclusion answers the paper's actual research problem instead of drifting into generic recommendations.
- Ensure the workflow's canonical final file is the finalized manuscript itself. A redirect note may exist as metadata, but not as the only final artifact.

## Undergraduate-thesis discipline

- Prefer chapter-complete structure over short article structure when the target artifact is `undergraduate thesis`.
- Include abstract, keywords, introduction, thematic or theoretical middle chapters, conclusion, and references.
- Management-oriented recommendations should remain subordinate to the scholarly argument and should not replace literature analysis.
- If the thesis is non-empirical, use conceptual analysis, mechanism analysis, comparative analysis, or framework building instead of simulated results.

## Readiness labels

- `submission-ready`: all required sections present, no obvious placeholders, references normalized, evidence boundaries explicit
- `thesis-ready`: chapter-complete, academically coherent, references normalized, final limitation language preserved
- `partial-finalization`: structure improved but blockers remain
- `not yet final`: critical sections or evidence still missing

## Common failure modes

- treating a polished draft as if it were already a final manuscript
- keeping TODO markers or meta commentary in the output
- adding fake empirical findings to create a stronger ending
- losing manuscript type discipline during last-pass rewriting
- ending with management advice while underdeveloping the literature or argument chapters
