# Scoping Playbook

Use this reference when a research request is too broad to execute directly.

## Narrowing sequence

1. Identify the real goal.
2. Convert the goal into 2-4 research questions.
3. Add practical boundaries.
4. Extract keyword clusters.
5. Recommend the next stage.

## Good boundary dimensions

- domain or field
- population or unit of analysis
- method family
- artifact type
- time window
- venue or literature source constraints

## Good scoping outputs

- narrow enough for search or drafting
- broad enough to remain meaningful
- transparent about what was inferred

## Failure modes

- keeping the topic at slogan level
- generating too many questions
- mixing problem framing and method choice too early
- using vague boundaries such as `cutting-edge` or `important`

## Handoff guidance

- If the next step is evidence collection, route to `paper-tracker`.
- If the user already has a corpus and wants synthesis, route to `literature-review`.
- If the user already knows the gap and wants a study plan, route to `research-design`.
