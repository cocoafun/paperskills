# Revision Playbook

Use this reference when turning feedback into an execution plan.

## Core sequence

1. Normalize the comments.
2. Separate major concerns from minor or editorial comments.
3. Map each issue to a section, evidence need, and action type.
4. Prioritize by decision risk and dependency.
5. Draft both paper changes and response-letter strategy.

## Action types

- repositioning
- rewrite for clarity
- add baseline or comparison
- add robustness evidence
- narrow overclaimed contribution
- acknowledge limitation explicitly

## Prioritization logic

Highest priority:

- issues that threaten acceptance
- issues requiring new evidence or experiments
- issues that affect multiple sections

Lower priority:

- local clarity fixes
- notation cleanup
- formatting and editorial polish

## Output discipline

The plan should separate:

- what to change in the paper
- what to say in the response letter
- what cannot be fixed before the deadline

## Failure modes

- treating comments as independent when they share one root issue
- promising experiments that are not feasible
- writing only a rebuttal strategy without a paper-change plan
